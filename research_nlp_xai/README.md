# Explaining and Auditing Emotion Recognition Models for Youth Mental Health Conversations

## Overview

This repository investigates the trade-offs between classical machine learning and transformer-based models for emotion recognition in youth mental health conversations.

While transformer architectures such as BERT achieve strong predictive performance, they often operate as black boxes. In sensitive domains such as mental health, understanding _why_ a model makes a prediction is often as important as the prediction itself.

This project evaluates model performance, interpretability, faithfulness of explanations, linguistic robustness, and error patterns using publicly available emotion datasets.

The work is designed as a reproducible NLP research project inspired by Responsible AI principles and human-centered machine learning.

---

## Research Questions

### RQ1

How much performance improvement do transformer models provide over classical machine learning methods for emotion recognition?

### RQ2

What linguistic features drive model predictions?

### RQ3

Are post-hoc explanations generated by transformer models faithful to their decision-making process?

### RQ4

How robust are emotion recognition models across different linguistic styles?

### RQ5

What trade-offs exist between predictive performance and interpretability?

## Explainability Analysis

### Classical Models

Intrinsic explanations using:

- Feature coefficients
- Token importance rankings

Example:

```
isolated   -> +2.14
lonely     -> +1.97
hopeless   -> +1.76
```

### Transformer Models

Post-hoc explanations using:

- SHAP
- LIME

Example:

```
isolated   +0.42
completely +0.12
today      +0.01
```

## Faithfulness Evaluation

To determine whether explanations truly reflect model behavior, we perform perturbation-based faithfulness testing.

### Deletion Test

Procedure:

1. Rank tokens by explanation importance.
2. Remove top-ranked tokens.
3. Recompute prediction confidence.
4. Measure confidence degradation.

### AOPC Evaluation

Area Over Perturbation Curve (AOPC) is used as a quantitative measure of explanation faithfulness.

A larger confidence drop indicates more faithful explanations.

## Linguistic Robustness Analysis

The datasets do not contain verified demographic labels.

Instead of demographic fairness claims, we evaluate robustness across linguistic styles.

### Linguistic Groups

- Standard English
- Slang-heavy English
- Informal Social Media Language
- Mixed Context Language

Metrics:

- False Positive Rate
- False Negative Rate
- Recall
- Selection Rate

Computed using Fairlearn.

## Error Analysis

Misclassified examples are manually reviewed and categorized.

### Error Categories

| Category         | Example                                      |
| ---------------- | -------------------------------------------- |
| Sarcasm          | "Great, another wonderful day of suffering." |
| Mixed Emotion    | "I'm scared but excited."                    |
| Slang            | "I'm cooked fr."                             |
| Long Context     | Multi-sentence emotional narratives          |
| Ambiguous Labels | Emotionally unclear samples                  |

This analysis helps identify systematic model weaknesses beyond aggregate metrics.
